### Lab 3: Основи Git (Podstawy Git)

[cite_start]**Zadanie 1[cite: 1]: Ініціалізація *repozytorium***
[cite_start]Створення папки, ініціалізація, додавання файлу до *staging* (poczekalnia) та перший *commit* (zatwierdzenie)[cite: 2, 3, 4, 6, 7].
```bash
mkdir projekt
cd projekt
git init
touch README.md
git add README.md
git commit -m "Initial commit"
```

[cite_start]**Zadanie 2[cite: 8]: Робота з файлами та історія**
[cite_start]Створення файлу, перші коміти та перегляд історії (*historia commitów*)[cite: 9, 10, 11, 12, 13, 14].
```bash
touch app.py
git add app.py
git commit -m "Dodano app.py"
git add app.py
git commit -m "Nowa funkcja"
git log
```

[cite_start]**Zadanie 3[cite: 15]: Виправлення коміту**
[cite_start]Створення коміту з помилкою у повідомленні та його редагування за допомогою `--amend`[cite: 16, 17, 18, 19].
```bash

git commit -m "błędna wiadomośćgit has"
git commit --amend -m "Poprawna wiadomość"
git log
```

[cite_start]**Zadanie 4[cite: 20]: Скасування коміту зі збереженням змін**
[cite_start]Створення коміту та його скасування (soft reset), зберігаючи зміни у файлах (у *katalogu roboczym*)[cite: 21, 22, 23, 24].
```bash
git add .
git commit -m "Test commit"
git reset --soft HEAD~1
git status
```

[cite_start]**Zadanie 5[cite: 25]: Безпечне скасування (Revert)**
[cite_start]Коміт з помилкою та його скасування без видалення історії (tworzy nowy commit cofający)[cite: 26, 27, 28, 29].
```bash

git log
```

### Lab 4: Запуск та базові команди терміналу

[cite_start]Ця лабораторна робота переважно описує структуру файлів README (Markdown) та перелік технологій для різних додатків (Simple Calculator, Todo Application, Student Management System тощо)[cite: 68, 69, 70, 83, 84, 95, 96, 98, 127]. [cite_start]Проте в інструкціях з інсталяції (*Installation*) наведено ключові команди терміналу для клонування та запуску[cite: 89, 92, 128, 131].

[cite_start]**Клонування віддаленого *repozytorium* та запуск програми на Python[cite: 90, 91, 93, 129, 130, 132]:**
```bash
git clone https://github.com/example/todo-app.git
cd todo-app
python app.py
```s

Це чудовий набір команд для підготовки. Ось детальне пояснення кожної з них із польською термінологією, щоб ти впевнено почувалася на *kolokwium*.

**Додавання та збереження змін (*Poczekalnia i zatwierdzanie*)**

* **`git add .`**
    Додає всі змінені та нові файли з робочого каталогу (*katalog roboczy*) до індексу/стадії підготовки (*poczekalnia* / *staging area*).
* **`git commit -m "Dodano 3 pliki"`**
    Створює новий коміт (*zatwierdzenie*) із файлами, які знаходяться в *poczekalni*. Прапорець `-m` дозволяє додати повідомлення до коміту.
* **`git commit --amend -m "Dodano 2 pliki"`**
    Зверни увагу на два тире перед `amend`. Ця команда змінює останній коміт. Дозволяє оновити повідомлення або додати до попереднього *zatwierdzenia* нові файли, якщо ти забула їх додати раніше.

**Історія змін (*Historia commitów*)**

* **`git log`**
    Показує повну історію комітів з усіма деталями (автор, дата, хеш, повідомлення).
* **`git log --oneline`**
    Скорочена версія історії. Кожен коміт відображається в одному рядку з коротким хешем і повідомленням.
* **`git log --graph --oneline --all`**
    Дуже корисна команда. Малює графічне дерево (*drzewo historii*), показуючи всі гілки (*gałęzie*), їх злиття та коміти в скороченому вигляді.

**Скасування змін (*Cofanie zmian*)**

* **`git revert <hash>`**
    Безпечне скасування. Створює новий коміт, який дзеркально скасовує зміни вказаного коміту. Історія не переписується (*nie usuwa historii*). 
    *Примітка: Твоє `git rever ^c` — це, ймовірно, просто перервана за допомогою `Ctrl+C` команда в терміналі.*
* **`git reset --hard HEAD~2`**
    Жорстке скасування. Пересуває покажчик поточної гілки на 2 коміти назад (можеш вказувати 1, 3, 4 тощо). **Увага:** ця команда безповоротно видаляє всі зміни з робочого каталогу (*katalog roboczy*) та *poczekalni* за ці коміти.

**Робота з репозиторіями та гілками (*Zdalne repozytoria i gałęzie*)**

* **`git clone <url>`**
    Завантажує (клонує) віддалене репозиторій (*zdalne repozytorium*, наприклад з GitHub) на твій комп'ютер.
* **`git branch <nazwa_gałęzi>`**
    Створює нову гілку (*gałąź* / *branch*), але не перемикає тебе на неї.
* **`git checkout <nazwa_gałęzi>`**
    Перемикає твій робочий каталог на вказану гілку (*przełączanie gałęzi*).
